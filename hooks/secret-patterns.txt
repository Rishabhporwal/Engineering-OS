# Shared secret VALUE patterns (ERE) — the single source for the blocking
# PreToolUse hook (on-secret-guard.sh) AND the commit gate (tools/secret_scan.py).
# These match secret *values* (prefix + realistic length/entropy), NOT bare
# prefixes — so documentation that mentions `shpss_…` as an example does not trip.
#
# Covers the six classes that actually leaked in incident O1 (Google OAuth,
# Shopify secret + PAT, Meta long-lived, DB connection-string password, app pw)
# plus common cloud/provider keys. One pattern per line; lines starting with # ignored.
#
# Add a new provider's key format here and BOTH the hook and the scanner pick it up.
